# File Descriptions

This document explains what each file in the Variable Dictionary project does.

---

## Root Directory

### `index.html`

The main HTML entry point for the application. It sets up the basic document structure, loads Google Fonts (JetBrains Mono for code and Source Sans 3 for UI text), applies minimal global CSS resets, and includes a root `<div>` where the React application mounts. Vite uses this file as the entry point during development and build.

### `package.json`

The npm configuration file that defines the project metadata, dependencies, and scripts. It lists React 19 as the main dependency and includes development tools like Vite (build tool), ESLint (code linting), and gh-pages (deployment to GitHub Pages). Key scripts include `dev` for local development, `build` for production builds, and `deploy` for publishing to GitHub Pages.

### `vite.config.js`

Configuration file for Vite, the build tool and development server. It enables the React plugin for JSX transformation and sets the base URL path to `/variable-dictionary/` for correct asset loading when deployed to GitHub Pages. If your repository has a different name, this path should be updated accordingly.

### `eslint.config.js`

ESLint configuration that defines code quality rules for the project. It extends recommended JavaScript rules and adds React-specific plugins for hooks validation and React Refresh compatibility. The configuration ignores the `dist` folder and allows unused variables that start with uppercase letters or underscores.

### `.gitignore`

Specifies files and directories that Git should ignore. Currently excludes `node_modules` (installed dependencies), `dist` (build output), and `.DS_Store` (macOS system files) from version control.

### `README.md`

Project documentation providing setup instructions, deployment guide, and usage information. It explains how to run the app locally, deploy to GitHub Pages, add or edit variables, and includes the project structure overview.

## Source Directory (`src/`)

### `src/main.jsx`

The JavaScript entry point that bootstraps the React application. It imports React and ReactDOM, then renders the main `App` component inside React's StrictMode (which helps identify potential problems during development). This file mounts the application to the `#root` element defined in `index.html`.

### `src/App.jsx`

The main application component containing all the UI logic and presentation. This single-file React component includes:

- **Constants**: Color mappings for variable types (wage, capital, benefits, demographics) and descriptions for CBS source tables.
- **Small Components**: `TypeBadge` (colored label for variable types), `InputPill` (clickable tags for input variables), and `CodeBlock` (syntax-highlighted Python code with copy functionality).
- **VariableDetail**: The right-panel component that displays comprehensive information about a selected variable including its definition, type, source tables, notebook reference, construction logic, input dependencies, and Python code.
- **App**: The main component managing application state (selected strategy, search query, type filter, selected variable), rendering the header with strategy tabs, the left sidebar with searchable/filterable variable list, and the detail panel.

### `src/data/variables.json`

The data file containing all variable definitions organized by research strategy. Each strategy (e.g., "Strategy 2", "Strategy 4") maps to an array of variable objects. Each variable includes:

- `name`: The variable identifier used in code
- `definition`: Human-readable description
- `type`: Category (wage, capital, benefits, demographics, or combined)
- `source`: CBS source table(s) used
- `notebook`: Associated Jupyter notebook file (or null)
- `logic`: Plain-English explanation of how the variable is constructed
- `inputs`: Array of input variable names this variable depends on
- `code`: Python code snippet showing the implementation
- `tag`: Optional label (e.g., "CBS source" for unmodified source variables)

Adding new variables or strategies is done by editing this file directly.

## Generated Directories (Not in Version Control)

### `node_modules/`

Contains all installed npm packages and their dependencies. This directory is created when running `npm install` and should not be committed to version control.

### `dist/`

Contains the production build output generated by `npm run build`. This is what gets deployed to GitHub Pages. It includes optimized, minified JavaScript and assets.
